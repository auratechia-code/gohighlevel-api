# Associations: Custom Relations

Manage relationships between different CRM objects (e.g., Contacts, Opportunities, Custom Objects).

## 1. Association Mapping

### Create Association
Define a relationship between two entities.

- **Endpoint:** `POST /associations/`

> [!TIP]
> **Official Docs:** [Create Association](https://marketplace.gohighlevel.com/docs/ghl/associations/create-association)

#### Example Requests
```bash
curl --request POST \
  --url https://services.leadconnectorhq.com/associations/ \
  --header 'Authorization: Bearer <TOKEN>' \
  --header 'Version: 2021-07-28'
```

```javascript
const axios = require('axios');

const options = {
  method: 'POST',
  url: 'https://services.leadconnectorhq.com/associations/',
  headers: {
    'Authorization': 'Bearer <TOKEN>',
    'Version': '2021-07-28'
  }
};

axios.request(options).then(function (response) {
  console.log(response.data);
}).catch(function (error) {
  console.error(error);
});
```

```python
import requests

url = "https://services.leadconnectorhq.com/associations/"

headers = {
    "Authorization": "Bearer <TOKEN>",
    "Version": "2021-07-28"
}

response = requests.request("POST", url, headers=headers)

print(response.text)
```

```php
<?php

$curl = curl_init();

curl_setopt_array($curl, [
  CURLOPT_URL => "https://services.leadconnectorhq.com/associations/",
  CURLOPT_RETURNTRANSFER => true,
  CURLOPT_CUSTOMREQUEST => "POST",
  CURLOPT_HTTPHEADER => [
    "Authorization: Bearer <TOKEN>",
    "Version: 2021-07-28"
  ],
]);

$response = curl_exec($curl);
$err = curl_error($curl);

curl_close($curl);

if ($err) {
  echo "cURL Error #:" . $err;
} else {
  echo $response;
}
```
- **Method:** `POST`
- **Scopes:** `associations.write`
- **Headers:** 
  - `Version: 2021-07-28`
  - `Authorization: Bearer <TOKEN>`

### Body Parameters
| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `locationId` | String | Yes | ID of the location. |
| `fromId` | String | Yes | Source object identifier. |
| `toId` | String | Yes | Target object identifier. |
| `type` | String | Yes | Type of relationship (e.g., `primary`, `secondary`). |

## 2. Records & Relations (Linking)

### Create Relation between Records
Link two specific record instances (e.g., link a specific Contact to an Opportunity).

- **Endpoint:** `POST /associations/relations`


#### Example Requests

## Error Catalog
| HTTP | Code | Message |
|------|------|---------|
| 400 | `invalid_input` | Irrelevant or missing IDs. |
| 401 | `unauthorized` | Missing or invalid Bearer token. |
| 404 | `not_found` | One of the record IDs or association keys does not exist. |
| 409 | `conflict` | Association already exists. |
| 422 | `unprocessable_entity` | Recursive or invalid object pairing. |
